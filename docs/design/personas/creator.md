# The Creator

> **Navigation**: [Design Index](../index.md) > [Personas](./index.md) > Creator

**Archetype:** The Skill Author Wanting Distribution

> *"I built something useful. I want others to find it."*

---

## Demographics

- Advanced Claude Code user (top 10%)
- Has created 1+ custom skills
- Active in developer communities
- Values recognition for contributions

---

## Goals

| Functional | Emotional |
|------------|-----------|
| Get discovered by relevant users | Feel recognized for work |
| Understand what users want | Connection with community |
| Improve based on feedback | Pride in helping others |
| Build reputation as author | Professional credibility |

---

## Fears and Anxieties

- Building something nobody finds
- Negative reviews or low scores
- Being compared unfavorably to alternatives
- Maintaining something that becomes popular

---

## Current Behavior and Workarounds

- Promotes skills on Twitter and dev forums
- Writes blog posts about skill creation
- Watches GitHub stars anxiously
- Responds to every issue and PR

---

## Trust Triggers (What Builds Confidence)

- Clear, fair scoring criteria
- Access to usage analytics
- Feedback mechanisms that feel constructive
- Path to improving discoverability
- Community recognition systems

---

## Delight Triggers (What Creates Joy)

- First install notification from a stranger
- Positive review or testimonial
- Seeing skill recommended in relevant contexts
- Download milestone celebrations
- User success stories featuring their skill

---

## Preferred Entry Points

1. **Author dashboard** with analytics
2. **Public profiles** showcasing their work
3. **Embeddable badges** for GitHub READMEs
4. **Terminal** for testing discoverability

---

## Red Flags That Cause Abandonment

- Opaque scoring with no improvement guidance
- No visibility into usage patterns
- Popular skills always ranking higher
- No author identity or profile
- Feedback that feels like criticism without help

---

## Design Implications

### For Discovery Features
- Show how skills are being discovered
- Provide search ranking insights
- Display related skill comparisons

### For Publishing
- Clear quality score preview before publish
- Specific improvement suggestions
- Simple submission process with status tracking

### For Analytics
- Real-time install notifications
- Usage patterns and trends
- Feedback aggregation and sentiment
- Discoverability insights

### For Community
- Author profile pages
- Skill portfolio display
- Recognition badges and milestones
- User success story collection

### For Ongoing Experience
- Maintenance reminders for popular skills
- Compatibility alerts for new Claude versions
- Community interaction tools

---

**Motivation:** "Recognition for my work"
**Fear:** Obscurity, unfair scoring
**Entry Point:** Author dashboard, public profiles
**Trust Trigger:** Fair scoring, feedback mechanisms
